## 2. Modèle de données

### 2.1 Nouvelle table `consentements`

```python
# backend/app/models/db_models.py

class Consentement(Base):
    """Trace immuable de tous les consentements (CGV, rétractation, RGPD biométrique).

    Append-only : une révocation crée une nouvelle ligne avec retire_le rempli,
    on ne MODIFIE jamais une ligne existante (preuve légale).
    """
    __tablename__ = "consentements"

    id = Column(String, primary_key=True, default=_new_uuid)
    utilisateur_id = Column(String, ForeignKey("utilisateurs.id"), nullable=True, index=True)
    email = Column(String, nullable=True, index=True)  # cas non authentifié
    type_consentement = Column(String, nullable=False, index=True)
    # 'cgv_checkout' | 'renonciation_retractation' | 'rgpd_biometrique' | 'rgpd_ia'
    accepte = Column(Boolean, nullable=False, default=True)
    version_texte = Column(String, nullable=False)  # ex: "cgv-v1.2-2026-05-19"
    ip = Column(String, nullable=True)
    user_agent = Column(Text, nullable=True)
    contexte = Column(Text, nullable=True)  # JSON : { plan, session_id, ... }
    accorde_le = Column(DateTime(timezone=True), nullable=False, default=_utcnow, index=True)
    retire_le = Column(DateTime(timezone=True), nullable=True)

    __table_args__ = (
        Index("idx_consentements_user_type", "utilisateur_id", "type_consentement"),
        CheckConstraint(
            "type_consentement IN ('cgv_checkout', 'renonciation_retractation', "
            "'rgpd_biometrique', 'rgpd_ia')",
            name="ck_consentements_type",
        ),
    )
```

## 7. Pièges à éviter

1. **Cases pré-cochées** : `defaultChecked` interdit. Les states React doivent démarrer à `false`. Tester explicitement.
2. **Conservation localStorage seule** : non opposable juridiquement. **Toujours** persister en DB.
3. **Modification en place d'une ligne consentement** : interdit. Append-only → créer une nouvelle ligne ou remplir `retire_le`.
4. **IP derrière Traefik** : utiliser `X-Forwarded-For` correctement (premier élément), pas `request.client.host` brut. Vérifier la config Traefik (header forwarding).
5. **Non-régression Stripe** : la fenêtre de 10 min doit être configurable (`CONSENT_FRESHNESS_MINUTES`) pour éviter de bloquer en cas de pause utilisateur. Tester edge case.
6. **Versions de texte** : si CGV change, **invalider** les consentements antérieurs (logique : `version_texte_actuelle != version_enregistree` → re-consentement). À documenter mais hors P1.3.
7. **Webhook Stripe** : ne PAS bloquer le webhook si consentement manquant a posteriori — le paiement est déjà fait. La vérif est UNIQUEMENT à la création de session.
8. **Compte non authentifié** : pour le checkout anonyme, le lien `email → utilisateur_id` se fait au webhook `customer.created`. Prévoir une migration paresseuse des lignes `consentements` (matcher par email puis remplir `utilisateur_id`).
9. **Modal RGPD déjà intégré** dans RestoreClient/AnimateClient : ne pas casser le contrat existant du hook (`requireConsent: () => Promise<boolean>`). Étendre, pas remplacer.
10. **Tests non-régression** : faire tourner `pytest backend/tests/` complet ET `npm test` côté frontend avant merge. CI doit échouer sinon.
11. **RGPD article 7.3** : retrait aussi facile que l'octroi. Le bouton "Révoquer" doit être visible sans friction au dashboard.
12. **Pas de mention de fournisseurs tech** dans les textes publics (CGV, modal) → "notre prestataire de paiement" et non "Stripe", "notre IA" et non noms de modèles.
